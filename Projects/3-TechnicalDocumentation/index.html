Introduction

CSS, short for Cascading Style Sheets, is a cornerstone technology of the web. It is used to style HTML documents and provides control over the layout, color, typography, and overall appearance of web pages.

CSS is designed to separate content (HTML) from presentation, allowing developers to apply consistent styles across multiple pages easily.

CSS Syntax

CSS is structured with rulesets. Each ruleset consists of a **selector** and a **declaration block**: 

selector {
    property: value;
    property: value;
}

For example:

h1 {
    color: blue;
    font-size: 20px;
}

Selectors

Selectors are used to target HTML elements for styling. Types of selectors include:

  • Universal Selector: Targets all elements (e.g., * {}).
  • Type Selector: Targets specific elements (e.g., h1, p).
  • Class Selector: Targets elements with a specific class (e.g., .classname).
  • ID Selector: Targets a specific element with an ID (e.g., #idname).
  • Pseudo Selector: Targets a special state of an element. (e.g. :hover applies style when an element is hovered over and :nth-child(n) targets the nth child of an element).

CSS Variables

CSS Variables allow developers to define reusable values in their stylesheets.

For example, a variable is defined like that:

:root {
    --main-color: #3498db;
    --padding-size: 10px;
}

And called like that:

div {
    color: var(--main-color);
    padding: var(--padding-size);
}

Box Model

The box model describes how elements are wrapped with content, padding, borders, and margins:

  • Content: Holds the elementâ€™s text or child elements. The size can be controlled with properties like width and height.
  • Padding: Adds space inside the element between the content and border. Use properties like padding or individual padding sides.
  • Border: Encases the content and padding, styled using properties like border, border-width, border-color, and border-radius.
  • Margin: Adds space outside the element, separating it from neighboring elements. Control it with properties like margin or individual margin sides.

Flexbox

Flexbox is a CSS layout model designed for flexible and efficient arrangement of items within a container, even when their size is unknown or dynamic.

Key FlexBox Properties:

  1. Container Properties:
    • display: Set to flex or inline-flex to enable flex context.
    • flex-direction: Determines the direction of the main axis:
      • row (default): Items are placed from left to right.
      • row-reverse: Items are placed from right to left.
      • column: Items are placed from top to bottom.
      • column-reverse Items are placed from bottom to top.
    • justify-content: Aligns items along the main axis:
      • flex-start (default): Items are packed at the start.
      • flex-end: Items are packed at the end.
      • center: Items are centered.
      • space-between: Items have equal space between them.
      • space-around: Items have equal space around them.
      • space-evenly: Items have equal space distributed.
    • align-items: Aligns items along the cross axis:
      • stretch (default): Items stretch to fill the container.
      • flex-start: Items are aligned at the start of the cross axis.
      • flex-end: Items are aligned at the end.
      • center: Items are centered.
      • baseline: Items align based on their text's baseline.
    • align-content: Aligns a flex container's lines when there are multiple lines. Works similarly to justify-content, but applies to cross-axis lines.
  2. Item Properties:
    • order: Controls the order of items. Default is 0. Lower values are placed first.
    • flex-grow: Determines how much an item will grow relative to others. (e.g. If two items have flex-grow: 1 and flex-grow: 2, the second item will take twice as much space).
    • flex-shrink: Specifies how items shrink relative to others when space is limited. (e.g. flex-shrink: 0 prevents shrinking).
    • flex-basis: Specifies the initial size of an item before space is distributed.
    • align-self: Overrides align-items for a specific item.

Grid Layout

CSS Grid is a powerful tool for creating two-dimensional layouts. It allows you to define rows and columns with ease.

Basic properties include:

  • display: Set to grid.
  • grid-template-rows: Defines the row structure.
  • grid-template-columns: Defines the column structure.

CSS Positioning

CSS positioning determines how elements are placed in the document flow and how they relate to other elements or the viewport.

The positioning is controlled using the position property with several values, each serving specific use cases:

  • Static (default): The default value for all elements. Elements are placed according to the normal HTML document flow. Top, right, bottom, and left properties have no effect.
  • Relative: Positions the element relative to its normal static position, while not affecting the position of neighboring elements. Moves the element based on the top, right, bottom, or left properties.
  • Absolute: Removes the element from the normal document flow, and positions it relative to its closest positioned (non-static) ancestor or the viewport if no ancestor is positioned. Other elements act as if it doesnâ€™t exist.
  • Fixed: Positions the element relative to the viewport, and stays fixed in the same spot even during scrolling. It's commonly used for sticky headers or footers.
  • Sticky: A hybrid of relative and fixed. The element is treated as relative until a specified scroll position is reached, after which it behaves like fixed. Requires a defined scroll container (e.g., overflow: auto on a parent).

Animations

CSS animations bring life to web pages by allowing elements to transition between states over time.

Key properties include:

  • @keyframes: Defines the animation steps.
  • animation-name: Specifies the keyframe.
  • animation-duration: Duration of the animation.
  • animation-delay: Defines a delay before the animation starts.
  • animation-iteration-count: Specifies how many times an animation will run.
  • animation-timing-function: Controls the speed curve of an animation (e.g., ease, linear, ease-in-out).

For example, an animation is defined like that:

@keyframes slideIn {
    from {
        transform: translateX(-100%);
        opacity: 0;
    }
    to {
        transform: translateX(0);
        opacity: 1;
    }
}

And used like that:

div {
    animation-name: slideIn;
    animation-duration: 2s;
    animation-timing-function: ease-in-out;
}

Media Queries

Media queries allow developers to apply CSS based on the device's screen size or other characteristics.

Example:

@media (max-width: 600px) {
    body {
        background-color: lightgray;
    }
}

They can be combined to handle different scenarios using the and and or keywords:

@media (min-width: 600px) and (max-width: 1200px) {
    body {
        background-color: lightblue;
    }
}
